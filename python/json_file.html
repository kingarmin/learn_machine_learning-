استفاده از JSON

آیا تابه‌حال به این فکر کرده‌اید که چگونه و با چه ساختاری می‌توان اطلاعات را ذخیره کرد و به‌سرعت به اشتراک گذاشت؟ به نظر می‌رسد که همیشه برای توسعه‌ی نرم‌افزارها، به دنبال راهی برای انتقال اطلاعات به‌صورتی ساده، قابل‌فهم و سریع هستیم. این هدف، به‌آسانی با JSON محقق می‌شود. JSON که مخفف JavaScript Object Notation است، یک ساختار سبک و قابل‌خواندن برای انتقالِ داده است که همه‌جا مورداستفاده قرار می‌گیرد، از برنامه‌نویسی تا مبادله‌ی اطلاعات در وب‌سایت‌ها. برنامه‌ای که شما در پایتون می‌نویسید، ممکن است با برنامه‌های دیگری که از تکنولوژی‌ها و زبان‌های برنامه‌نویسی دیگری استفاده می‌کنند، در ارتباط باشد؛ بنابراین برای انتقال داده‌ها، باید از زبان مشترکی استفاده کنید که برنامه‌های دیگر نیز آن را بفهمند. JSON حکم همین زبان مشترک را دارد.

فرض کنید می‌خواهیم اطلاعاتی از یک شخص را در یک فایل ذخیره کنیم. این اطلاعات شامل نام، سن، آدرس و شاید چیزهای دیگری است. این‌جاست که JSON به کمک ما می‌آید. داده‌هایی که ساختار JSON دارند را می‌توان در فایلی با پسوند .json ذخیره کرد. در این مثال، ما اطلاعات شخص را در فایل info.json ذخیره کرده‌ایم.

info.json
{
  "firstName": "Mohamadreza",
  "lastName": "Kariminejad",
  "isAlive": true,
  "age": 18,
  "address": {"city": "Yasuj", "village": "Karim-Abad"},
  "children": [],
  "wife": null
}
JSON

ساختار مثال بالا را در موارد زیر بررسی می‌کنیم.

قالب JSON🔗

برای مقداردهی در JSON از : به جای ‍= استفاده می‌شود. به‌طور کلی، ساختار JSON بسیار شبیه به لغت‌نامه‌ها در پایتون است؛ به این معنی که داده‌ها به‌صورت کلید و مقدار (key & value) ذخیره می‌شوند. کلیدها باید حتماً از جنس رشته باشند و مقدارها، مانند پایتون، انواع داده‌های مختلفی دارند. مهم‌ترین انواع داده‌‌های JSON را در زیر آورده‌ایم.

رشته (string) و عدد🔗

هر دو نوع، مشابه رشته و عدد در پایتون هستند؛ با تفاوت‌هایی کوچک:

  • رشته‌ها باید حتماً بین دو " قرار بگیرند و نمی‌توان از ' استفاده کرد. برای مثال، "Ali" صحیح است و 'Ali' پذیرفته نمی‌شود.
  • اعداد می‌توانند شامل E یا e باشند؛ به عبارت دیگر، JSON از اعداد علمی نیز پشتیبانی می‌کند. برای مثال، 3.7E-5 در JSON پذیرفته می‌شود.

منطقی (boolean)🔗

این نوع داده دقیقاً مشابه bool در پایتون است با این تفاوت که در پایتون True و False با حرف بزرگ شروع می‌شوند ولی در JSON به‌صورت true و false هستند.

آرایه (array)🔗

این نوع داده دقیقاً مشابه لیست در پایتون است؛ یعنی چند مقدار درون [] قرار می‌گیرند و با استفاده از , از هم جدا می‌شوند. دقت کنید که نوع مقادیر آرایه می‌تواند با هم متفاوت باشند. برای مثال، ["orange", 2, true] آرایه‌ای صحیح در JSON است.

لغت‌نامه (dictionary)🔗

این نوع داده دقیقاً مشابه لغت‌نامه در پایتون می‌باشد. کلید‌های آن حتماً باید از نوع رشته باشد و همان‌طور که گفته شد، رشته‌ها را نمی‌توان بین دو ' قرار داد و باید بین دو " قرار بگیرند. برای مثال، {"fruit": "orange", "amount": 2} لغت‌نامه‌ای صحیح در JSON است.

استفاده از JSON در پایتون🔗

پایتون برای کار با ساختار JSON کتابخانه‌ی استانداردی دارد که با استفاده از import json می‌توان آن را به برنامه‌ی خود اضافه کرد. با استفاده از این کتابخانه، می‌توانیم نوع داده‌های پایتونی را به JSON و برعکس تبدیل کنیم. در ادامه، برخی توابع پرکاربرد این کتابخانه را بررسی می‌کنیم.

تابع load🔗

همان‌طور که گفته شد، ساختار کلی JSON بسیار شبیه به لغت‌نامه‌ها در پایتون است؛ بنابراین باید بتوان آن‌ها را به‌راحتی به یکدیگر تبدیل کرد. تابع load، یک فایل با فرمت .json را ورودی می‌گیرد و آن را به یک لغت‌نامه‌ی پایتونی تبدیل می‌کند. فایل info.json را که در ابتدای درس‌نامه ساختیم، در نظر بگیرید. با استفاده از دستورهای زیر، محتوای این فایل را در قالب یک لغت‌نامه در متغیر data در پایتون ذخیره می‌کنیم.

>>> import json
>>> with open('info.json') as info:
...     data = json.load(info)
...
>>> data['firstName']
'Mohamadreza'
>>> data['address']['city']
'Yasuj'
Python

نکته: از آن‌جا که کد بالا در پایتون نوشته شده‌است و داده‌های فایل info.json به داده‌های پایتونی تبدیل شده‌اند، استفاده از ' برای رشته‌ها مجاز است.

تابع dump🔗

این تابع، برعکس تابع قبلی عمل می‌کند و داده‌های پایتونی را به ساختار JSON تبدیل و در فایلی ذخیره می‌کند. برای مثال، فرض کنید می‌خواهیم اطلاعات یک فرد دیگر را در فایل دیگری ذخیره کنیم.

>>> import json
>>> data = {"firstName": "Maryam",
...         "lastName": "Amiri",
...         "age": 23}
...
>>> with open('info2.json', 'w') as file:
>>>    json.dump(data, file)
Python

بدین ترتیب، اطلاعات موجود در لغت‌نامه‌ی پایتونی data در فایل info2.json ذخیره می‌شود.

تابع loads🔗

داده‌های JSON ممکن است لزوماً در یک فایل قرار نداشته باشند. گاهی ما این اطلاعات را از جای دیگری به‌صورت رشته دریافت می‌کنیم. این اتفاق، هنگام تبادل داده درون یک نرم‌افزار یا وب‌سایت بسیار رخ می‌دهد. در این صورت، برای تبدیل JSON به داده‌های پایتونی می‌توانیم از تابع loads استفاده کنیم. حرف s در این‌جا نشان‌دهنده‌ی string است.

این تابع داده‌های JSONی را در قالب یک رشته ورودی می‌گیرد و در خروجی آن را به پایتون ترجمه می‌کند و در قالب لیست، لغت‌نامه یا دیگر اشیاء متناظر در پایتون خروجی می‌دهد. به مثال زیر دقت کنید.

>>> import json
>>> json.loads('2')
2
>>> json.loads('{"people": ["ali", "mamad", "reza"]}')

اگر ورودی‌ تابع کاملاً طبق استاندارد JSON نباشد، استثنای JSONDecodeError داده می‌شود که مشخص می‌کند ورودی معتبر نیست و نمی‌توان آن را به داده‌های پایتونی تبدیل کرد. برای مثال اگر سعی کنیم 'ali' را به داده‌ی پایتون ترجمه کنیم، خطا رخ می‌دهد. کافی است آن را به‌صورت "ali" بنویسیم تا به یک قالب معتبر تبدیل شود.

تابع ‍‍dumps🔗

این تابع برعکس تابع قبلی است و یک ورودی از جنس داده‌های پایتونی می‌گیرد و آن را به شکل رشته‌ای شامل داده‌های JSONی درمی‌آورد. به مثال زیر دقت کنید.

>>> import json
>>> json.dumps([1, 2, 3])
'[1, 2, 3]'
>>> json.dumps({"key": 2, "list": [12, 'ali']})
'{"key": 2, "list": [12, "ali"]}'
>>> json.dumps([{"key": "value"}])
'[{"key": "value"}]'
>>> json.dumps(2)
'2'
